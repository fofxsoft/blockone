Product Editor Overview

The Product Editor will cover the fundimentals of Angular and the Encompass API.

By the end of this tutorial you will be able to do the following:

  • Use built-in Angular directives to show and hide elements and display lists of products.
  • Create Angular components to display product details and show an array of products.
  • Use one-way data binding for read-only data.
  • Add editable fields to update a model with two-way data binding.
  • Bind component methods to user events, like keystrokes and clicks.
  • Enable users to select a product from a master list and edit that product in the details view.
  • Create a shared service to assemble the products from the Encompass API.
  • Use routing to navigate among different views and their components.

This tutorial also uses the Material UI Design Concepts. These concepts ensures that we create a uniform user experience across mutliple platforms. This tutorial will cover the tools needed to achieve this.

What you'll build

Here is what this app will look like. Starting with the Dashboard.

From there users will be able to click on Products in the menu on the left. This will display the product list.

Users will be able to click on any product in the product list. This will display the product detail.

Users will be able to edit, name, proce, case price, and three barcode fields. If the product doesn't have a supplier product number, users will be able to enter it. Once the supplier product number is set, it will not be editable.
