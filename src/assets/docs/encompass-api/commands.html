API Commands

Let's get familiar with the API Commands table. This is where the definition of your API lives.

Navigating to the API Commands Table

Log in to your publisher site. For this example we will use test.encompass8.com.

In the search field enter API Commands. You will be taken to TableView.aspx?TableName=APICommands.

TableView.aspx?TableName=APICommands

Get to Know the API Command Fields

Click the Add API Command Button.

TableView.aspx?Action=Add&TableEdit=APICommands

A brif explination of each field;

Fields marked with a * are required. And any field you filter on will be automatically populated.

  • Publisher ID Set this to your publisher name.

  • Command Name This is the name of the command, you will use this in your application.

  • URL This is the URL inside Encompass. This tells the API how to get to the data. For this tutorial we will always append Format=JSON.

  • Allowed Parameter Array By default Encompass will not allow any parameters. This is for security. This field "white-lists" parameters. When a parameter is passed to the API that is not "white-listed" the API call will fail.

  • Active This allows you to deactivate a command without deleting it.

  • Is Global This field allows you to install a command that doesn't need to be installed on every distributor site. Note: Many publishers might not see this field, because of permissions.

  • Description This is an open test field. Allows you to add a description to your command.

Typically you will not be making commands manually. Encompass has shortcuts in each system that supports API Commands.

Testing You API Commands

Because Encompass uses an "open ended" system and handles authentication for you, you do not need any authrization headers. This makes it very easy to test API commands with your browser, with an exception to importing data that uses POST.

Once you have created commands and installed them in your test site, you will be able to test and view the output by entering this URL;

https://api.encompass8.com/aspx1/API.aspx?Token=[YOUR_TOKEN_HERE]&EncompassID=[ENCOMPASS_ID_OF_TEST_SITE]&Command=[COMMAND_NAME]

When you install an API Command you will be given a sample link that you can use for testing.
