Get Product Information Command

For the Product Editor we will need a count of inventoried products, non inventoried products and a total product count. You can't do this with a table view, so we will need to use a Fusion Report to get this information.

Create the Product Information Report

In your Encompass test site, navigate to a new Fusion Report. Enter Fusion Report in the search field.

FusionEdit.aspx

In the Tables section, click Add Table -> Add Table.

Enter Products in the Table field. Then click Select.

Now click Add Table -> Add Related Table.

Check Product Types, and then click Select.

Click on the Data tab.

In the Group By section under Fields enter ProductTypes - Track Inventory in the first position.

In the Values section enter Products - Product ID in the first position.

Change the value name from Product ID to Product Count.

Change the aggregate to Count. This is the drop down after the field.

Click Create Report.

Save the Fusion Report

You should now have a simple report that counts products by the track inventory flag.

FusionView.aspx

Click Save Report As.

  • Enter Product Counts in the Report Name field.

  • Set the Format to JSON.

Click Save.

Create the productInfo Command

Now that your report is saved, you now have access to the Share menu.

For this example, we will use the automated Share system to create the API Command. The Share system is available to tables too, but practice making commands manually helps us understand how the API works.

Click Share -> API Command.

  • Enter ProductInfo in the Command Name field.

  • Set the Publisher field to your publisher name.
  • Change the Page field from FusionView.aspx?ReportID=1234 to FusionView.aspx?ReportID=1234&Format=JSON
    Remember we need to add Format=JSON to your API Commands.

Click Save

What's Next?

You're almost done! You have created a set of API Commands that will provide data for the Product Editor tutorial. Now all you have to do is install the commands on the test site.
